
Solucion de problemas repetitivos que normalmente se ve en el codigo

## Clasificacion de patrones
* Creacional 
* Estructural 
* Conductual 
## ¿Por que aprender patrones?
Es usado para programar, se resume la explicacion para un error con patrones requeridos evitando explicacion.

## Ejemplos de patrones:

### Builder(Creacional)

Problema con el contrsutor utilizando muchos parametros 
Constructor (Creativo)

Constructor es un patrón de diseño creacional que permite construir objetos complejos paso a paso. El patrón permite producir diferentes tipos y representaciones de un objeto utilizando el mismo código de construcción


Constructor (Creativo)

Imagina un objeto complejo que requiere una inicialización laboriosa, paso a paso, de muchos campos y objetos anidados. Dicho código de inicialización suele estar enterrado dentro de un constructor monstruoso con muchos parámetros. O peor aún, disperso por todo el código del cliente.
